## 预处理指令

在词法分析之前执行。行首（允许前导空白）以 `#` 开头的整行视为预处理行；**字符串字面量内部**的 `#` 不触发预处理。

### `#include`

```kval
#include "相对或文件名.kval"
#include <相对或文件名.kval>
```

**查找顺序：** 先在当前源文件所在目录查找；若无，再在包内 **`Kval/Lib/`** 下查找。`""` 与 `<>` 使用**相同**的查找规则。

### `#define`

对象式宏：把标识符替换为后续文本。行尾 `//` 起为注释，**不计入**替换文本。

```kval
#define NAME 替换文本
#define FLAG
```

`#undef NAME` 用于取消宏定义。

- 展开只作用于普通源码行，**不在**其它预处理行（如另一条 `#define`）中展开。
- 按「整词」替换；字符串 `"..."` 内的标识符不展开。
- 展开深度有上限；相互循环展开会报错。

### `#pragma once`

若被包含文件中出现过 `#pragma once`（大小写不敏感），则该路径在同一翻译单元内**至多展开一次**；之后再次 `#include` 同一文件时，得到空内容。

## 词法与注释

在**预处理完成之后**，对展开得到的源码进行词法分析。下列注释可出现在任意相邻记号之间；**字符串字面量**（`"..."` 与 `'...'`）**内部**的 `//`、`/*` 按普通字符处理，不视为注释起始。

### `//` 行注释

从 `//` 起直至该行换行符（不含换行符）之间的字符均被忽略，不产生记号。

```kval
int x;  // 行尾说明
// 整行都是注释
```

### `/*` `*/` 块注释

从 `/*` 起至其后**第一个** `*/` 为止（**不嵌套**：内部的 `/*` 不开启新块）之间的字符均被忽略，可跨多行。

```kval
int a;
/* 单行块注释 */
int b;
/*
  多行
  注释
*/
int c;
```

### 与预处理、`#define` 的关系

- `#define` **行尾**的 `//` 由预处理器剔除，**不计入**替换文本（见上文「`#define`」）。
- 若把 `/* ... */` 写在宏的替换文本中间，会随文本一起参与宏展开；需要注释整条 `#define` 时，请使用行尾 `//`，或把说明写在**非** `#` 开头的普通行上。

### 词法保留字总表（以 `Core/Parser/Lexer.py` 为准）

以下单词会被词法器识别为关键字记号（而非普通标识符）：

```kval
int float bool auto const static constexpr void string array
return delete varScopeOrder template typename
class struct public protected private code New operator roperator
global local builtins closure this base
true false
if elif else while for switch case break continue
throw try catch finally
import from as export namespace
```

### 复合运算符与标点记号（词法层）

词法器支持以下多字符记号：

```kval
** <<= >>= += -= *= /= %= ^= &= |=
|| && :: -> <= >= == != .=
```

其余一元/二元符号按单字符切分（如 `+ - * / % ^ & | ~ < > ( ) [ ] { } ; , : = . !`）。

## 变量

### 定义

**局部变量**

```kval
<类型> <变量名>
```

**带作用域前缀的变量**

内置作用域名（不可挪作自定义作用域标签）：

| 名称       | 含义       | 备注 |
| :--------- | :--------- | :--- |
| `builtins` | 内置作用域 | — |
| `global`   | 全局作用域 | — |
| `closure`  | 闭包作用域 | `closure{n}` 为第 n 层外围闭包 |
| `local`    | 局部作用域 | — |

语法：

```kval
<类型> <作用域>::<变量名>
```

可与初始化合并：

```kval
<类型> <变量名> = <值>
<类型> <作用域>::<变量名> = <值>
```

### 删除变量

```kval
delete <变量名>, ...
```

- `...` 表示可写多个变量名，逗号分隔。
- 删除不存在的名字不报错。
- 不能使用 `作用域::` 形式删除。对**非模块**闭包帧按顺序查找；若在某帧 `f_locals` 中找到该名则删除并 `break`（不会删到模块形帧）。

### 赋值

```kval
<变量名> = <值>
<作用域>::<变量名> = <值>
```

右侧类型须与变量声明类型一致，否则**编译期**报错。

### 复合赋值

支持将二元运算与赋值合一（左值须为裸变量或 `<作用域>::<名>`，不支持 `obj.attr +=` 等形式）：

```kval
<变量名> += <表达式>
<变量名> -= <表达式>
<变量名> *= <表达式>
<变量名> /= <表达式>
<变量名> %= <表达式>
<变量名> ^= <表达式>
<变量名> &= <表达式>
<变量名> |= <表达式>
<变量名> <<= <表达式>
<变量名> >>= <表达式>

<作用域>::<变量名> += <表达式>
// … 同上，其余复合运算符均可用于作用域限定形式
```

语义等价于先读左值、再与右侧做对应运算、再写回；**静态类型**在解析阶段检查：左值须已具类型（先 `int` / `string` / 作用域声明等），且右侧及运算结果类型须符合该左值类型（例如 `int` 仅与 `int` 做算术位运算；`string` 仅支持 `+=`（右侧 `string`）与 `*=`（右侧 `int`，重复））。

### 静态类型检查

在 `Parser.parse_source` 完成语法分析后，对模块做一次**静态**类型遍历（见 `Kval/Core/TypeCheck.py`）：变量须先声明再赋值；赋值与复合赋值与表达式中的 `int` / `string` 及模板形参等须一致；`return` 与函数签名一致；对泛型模板函数的**函数体**暂不深入检查，对其**调用**放宽实参匹配以便与 `<<…>>` 实例化前的语法一致。

### 裸名赋值、删除与「非跨函数」闭包

内层函数执行时，其栈帧的 **`f_back`** 指向**调用它的那一层**的运行帧（通常为直接外层函数）。当裸名赋值解析到**某一闭包捕获帧** `closure`，且 **`closure` 正是当前帧的 `f_back`**、且该帧**非模块形**（`f_locals` 非 `f_globals`）时，视为**非跨函数闭包**：允许对该帧中**已存在于 `f_locals` 的绑定**做裸名赋值，**即使**该名字未出现在该帧的 `declared_locals`（例如实现或工具注入的单元；一般用户代码仍应先 `int` 等声明）。

若目标闭包帧**不是**当前 `f_back`，则仍须满足该帧的 `declared_locals` 规则，否则报「未声明即赋值」类错误。

`delete <裸名>` 在闭包链上查找时同样跳过模块形帧；命中则删除（与是否 `declared_locals` 无关，以实现为准）。

### 读取

**裸名（默认解析顺序）**

```kval
<变量名>
```

默认解析顺序：`local` → `closure` → `global` → `builtins`。未找到时得到 `Unbound` 或报错，取决于运行时选项（如 `--skip-error`，若实现）。

**显式作用域**

```kval
<作用域>::<变量名>
```

## 声明修饰符与类型语法

> 本节仅给出语法形式（shape），不描述类型系统与运行时语义。

### 修饰符位置

```kval
const <类型> <变量名> = <表达式>;
constexpr <类型> <变量名> = <表达式>;
constexpr <返回值类型> <方法名>(<参数列表>) { ... }
static <类型> <变量名> [= <表达式>];
```

### 类型与关键字（语法层）

```kval
<类型> ::= int | float | bool | string | array | void | auto | <标识符类型名>
```

### `struct` 语法

```kval
struct <结构名> {
  <成员声明>...
}
```

### `varScopeOrder`（改写裸名查找）

以下语句必须出现在：该 `<变量名>` **第一次**以裸名参与表达式**之前**。只影响此后该名字的**自动（裸名）查找**；**不改变** `<作用域>::<变量名>` 的含义。

关键字 **`varScopeOrder`**；`:` 后有两种形式，可**并列多组**（组与组之间逗号分隔）。

**形式 A — 固定单一作用域**（冒号后**一个**作用域名，无括号）

```kval
varScopeOrder <变量名>: <作用域>, ...
```

- 裸名查找**仅**在该作用域中查找；无绑定即结束，**不回退**到默认四层顺序的其它层。
- 会清除该名字此前由本语句设置的「自定义链」。

**形式 B — 自定义查找链**（冒号后为括号内、用 `->` 连接的作用域序列）

```kval
varScopeOrder <变量名>: (<作用域> -> <作用域> -> ...), ...
```

- 从左到右依次尝试；当前格无绑定则试下一格。
- 全部无绑定则结束，**不再**使用默认四层解析顺序。
- 会清除该名字此前设置的「固定单作用域」规则。

## 方法

方法即带参数与返回类型的可调用单元；可出现在模块或类体等上下文中。

### 定义

```kval
<返回值类型> <方法名>(
  <位置参数类型> <位置参数名>, ..., <变长位置参数类型> *<变长位置参数名>, /,
  <任意传入方式的参数类型> <任意传入方式的参数名>, ..., *,
  <关键字参数类型> <关键字参数名>, ..., <变长关键字参数类型> **<变长关键字参数名>
) {
  <代码>
  return <返回值>;
}
```

- 返回值非 `void`：所有路径须能到达 `return`，否则编译报错。
- 返回值为 `void`：不产生可用的返回值。
- `/` 前与最后一个 `*` 后的段落均可省略；省略后参数按「任意传入方式」处理。
- 参数列表与调用处均可随意换行排版。

### 调用

```kval
<方法名>(
  <位置参数值>, ..., *<位置参数的序列对象>,
  <关键字参数名>=<关键字参数值>, ..., **<关键字参数的映射对象>
)
```

### 重载

**同一作用域**内，同名且**形参列表（签名）不同**的多个方法构成重载集；调用时由实参匹配唯一或最佳重载（歧义则报错，以实现为准）。

> **实现状态：** 已实现**非模板**同名函数的**重载集**：运行时与静态检查均按实参个数与关键字匹配唯一签名；无匹配或歧义时报错。模板函数仍按单独规则处理。

### 模板

使同一段定义可作用于多组类型或常量等模板实参：

```kval
template <<模板参数类型> <模板参数名>, ...>
<方法定义>
```

**排版：** `template ... >` 的结束符 `>` 与随后的**方法定义**（返回类型、方法名、`(` 起头的参数列表）须处于**不同物理行**，避免与比较运算符 `>` 混淆。

模板形参类型可为 **`typename`**，表示对应实参须为**类型名**。

**调用**（尖括号内为模板实参，圆括号内为调用实参）：

```kval
<方法名><<模板参数值>, ...>(<调用实参>)
```

模板实参列表可单行或多行。

## 表达式与运算符

### 逻辑条件运算

- **`!`**：单目逻辑非；优先级与同层的 `-`（取负）、`~`（按位取反）一致，且 **`!` 最先结合**（右结合的一元链）。  
- **`&&`**：逻辑与，**短路**：左操作数为假时不求值右操作数。  
- **`||`**：逻辑或，**短路**：左操作数为真时不求值右操作数。  

**优先级（从紧到松）：** 一元 `!` → 乘除取模与移位 → 加减 → 比较（`<` `<=` `>` `>=` `==` `!=`）→ **`&&`** → **`||`**。  

**真值约定**（与 `if` / `while` 一致）：`int` 非 0 为真、`0` 为假；`string` 非空为真、空串为假；`Unbound` / `null` 为假。`&&` / `||` 的求值结果为 **`int`**：`0` 或 `1`。

**实现说明：** 含 **`&&` / `||`** 的表达式在含 ASM 后端的函数中会**整函数回退到 AST 解释**，以保证短路语义；**`!`** 可在 ASM 路径中直接支持。

## 控制流

控制流语句用于按条件分支、循环与多路分支执行代码；一般写在方法体、`code` 段或其它允许语句的上下文中。

### `if` / `elif` / `else`

```kval
if (<条件>) {
  <代码>
}
elif (<条件>) {
  <代码>
}
else {
  <代码>
}
```

- `if` 必选；`<条件>` 为布尔语境下的表达式（以实现为准）。  
- `elif`、`else` 可选；可连续多个 `elif`。  
- `else` 至多一个，且须排在所有 `elif` 之后。  
- 各分支体须为 `{ ... }` 块（若以实现允许单语句省略块，以实现为准）。

### `while`

```kval
while (<条件>) {
  <代码>
}
```

在每次迭代**开头**求值 `<条件>`；为真则执行块，再重复；为假则离开循环。

支持 **`break`**（结束最内层 `while` / `for` / `switch` 的 `case`/`else` 块）与 **`continue`**（结束本轮、进入下一轮 `while` / `for`）。

### `for`

**形式一 — 类 C 的三段式**

```kval
for (<初始化>; <条件>; <步进>) {
  <代码>
}
```

- **`<初始化>`**：进入循环前执行一次；常见为变量定义与赋值，也可为赋值或空（以实现为准）。  
- **`<条件>`**：每次迭代**开头**求值；为假则结束循环。  
- **`<步进>`**：每次执行完循环体后执行。  
- 分号分隔三段；任一段是否允许省略以实现为准。

**形式二 — 区间迭代**

```kval
for (<元素类型> <元素变量> : <可迭代对象或迭代器>) {
  <代码>
}
```

- **`<可迭代对象或迭代器>`** 须满足文档「类 → 特殊协议」中的**可迭代对象**或**迭代器对象**约定。  
- 每轮迭代将当前元素绑定到 `<元素变量>`，在块内使用。

### `switch`

```kval
switch (<表达式>) {
  case (<case表达式>) {
    <匹配到时代码>
  }
  ...
  else {
    <未匹配到时代码>
  }
}
```

- 先求值 `switch` 的 **`<表达式>`**，再与各 `case` 括号内的 **`<case表达式>`** 做相等性比较（相等判定以实现为准）。  
- 匹配到的第一个 `case` 块执行；**`else`** 表示所有 `case` 均未命中时执行。  
- `case` 与 `else` 的块体须为 `{ ... }`。  
- 是否贯穿（fall-through）到后续 `case`、是否必须写 `else`，以实现为准。

## 类

### 定义

```kval
class <类名> {
  public:
  ...
  protected:
  ...
  private:
  ...
  code:
  ...
};
```

类体内的变量、方法、嵌套类声明为该类的**成员**（`code` 段除外，见下表）。

### 访问段与 `code`

| 名称        | 类别       | 含义 |
| :---------- | :--------- | :--- |
| `public`    | 访问控制   | 任意可见处可访问 |
| `protected` | 访问控制   | 仅本类及派生类内部 |
| `private`   | 访问控制   | 仅本类内部；未写访问段时默认按 `private` |
| `code`      | 程序段     | 类体展开时执行的可执行逻辑，**不是**普通成员块 |

建议在 `public` / `protected` / `private` 中只写成员相关声明；`code` 中可写任意语句。

### 特殊成员

> **实现状态：** 已实现类体内 **双目 `operator`**（如 `operator+`）：解析为成员函数并隐式增加 `this` 形参，运行时对左操作数为类实例的 `+` `-` `*` `/` `%` 分发到 `__op_add__` 等成员（见实现）。**构造调用**：在表达式位置支持 **`类名(实参…)`**；若类内**无**同名 `void 类名(…)`，则回退调用成员 **`New类名`**（由 **`任意类型 New<类名>(…)`** 语法糖生成）。**`New<类名>(…)`** 已接入解析器，成员名为 **`New` + 尖括号内标识符**（如 `NewPoint`）。`roperator` / `uoperator`、析构 `~T`、用户定义转换等**尚未**完整接入解析器；类体仍支持访问段、`code` 段与普通成员声明。

1. `(public) <任意类型> New<类名>(<函数签名>)` — 自定义实例的创建逻辑。  
2. `(public) void <类名>(<函数签名>)` — 构造函数。  
3. `(public) void ~<类名>()` — 析构函数。  
4. `(public) <任意类型> operator<双目操作符>(<任意类型> <另一操作数>)` — `this` 为**左**操作数的双目运算。

   | 操作符 | 含义 |
   | :----- | :--- |
   | `+` `-` `*` `/` `%` | 算术 |
   | <code>^</code> <code>&</code> <code>|</code> | 按位异或、与、或 |
   | `<<` `>>` | 左移、右移 |
   | `<` `<=` `>` `>=` `==` `!=` | 比较 |
   | `.` | 属性读取 |
   | `.=` | 属性写入 |
   | `[]` | 索引读取 |
   | `[]=` | 索引写入 |

5. `(public) <任意类型> roperator<双目操作符>(<任意类型> <另一操作数>)` — `this` 为**右**操作数的双目运算。  
6. `(public) <任意类型> uoperator<单目操作符>()` — 单目运算。

   | 操作符 | 含义 |
   | :----- | :--- |
   | `-` | 一元负号 |
   | `~` | 按位取反 |

7. `(public) <目标类型> operator <目标类型名>()` — 用户定义到 `<目标类型>` 的转换。

### 特殊协议

#### 可迭代对象

支持两种方式（运行时**先尝试方式一**，不可用时再尝试方式二）。

**方式一 — `iter` / `next`**

- 容器实例：`<迭代器类型> iter()`  
- 迭代器实例：`<任意类型> next()`

对 `iter()` 的返回值：若不是「迭代器对象」，则对其继续调用 `iter()`，直到得到迭代器，再按 `next()` 驱动迭代。

**方式二 — 下标与长度**

- 实例上：`<任意类型> operator[](int index)`  
- 且具备 `int length()` **或** `int length` 属性  

自 `0` 起递增下标，直至达到 `length` 所表示的结束条件。

#### 迭代器对象

须同时满足：

- `<迭代器类型> iter()` **返回自身**；  
- `<任意类型> next()` 在尚有元素时返回下一项，结束时抛出 **`StopIteration`**。

### `this` 指针

在实例方法内，`this` 为指向**当前实例**的指针，用于访问当前对象的成员或与自身相关的重载逻辑。

### 实例构造、方法调用与回收（实现说明）

本节说明**当前实现**与上文「目标语法」的对应关系，便于理解实例何时产生、`this` 在方法里指什么、以及何时被回收。

1. **构造**  
   - **`类名(实参…)`**（表达式位置）：运行时从已注册的类对象上 **浅拷贝** `members` 得到新实例，再优先调用 **`members` 中与类名同名的** `void 类名(…)` 构造函数；**若无**，则调用 **`members["New" + 类名]`**（即源码里写的 **`任意类型 New<类名>(…)`**，解析为成员名 **`New类名`**）。两种情况下均将 **`this` 绑定到该新实例** 后执行函数体。  
   - **`类名 变量;`** 且无初始化：对类类型当前实现为 **拷贝模板 `members`** 得到实例字典；**不保证**会调用同名构造函数或 `New类名`（与上一行显式 `类名(...)` 路径不同）。  
   - **声明语法**：在类体或模块中可写 **`<返回类型> New<目标类名>(形参…) { … }`**（`void` / `int` / 自定义类型等作返回类型均可）；解析器将成员登记为 **`New目标类名`**，与手写 **`NewPoint`** 等普通成员函数名等价。

2. **方法调用与 `this`**  
   - 文档称 `this` 为**指针**；实现上实例为带 `__kclass__` 与 **`members`** 的映射，**`实例.方法(…)`** 时运行内向 `KvalFunction` 注入接收者，使方法体内的 **`this` 指向本次调用的接收者**；成员读写走 **`members`**。语义上是「当前对象」，**不是** C/C++ 的裸指针与手动 `delete` 模型。

3. **析构与生命周期结束**  
   - 文档中的 **`void ~类名()`** 析构成员**尚未**接入运行时；**没有**在实例销毁前自动调用的析构钩子。  
   - **回收**：实例由宿主（CPython）管理，无引用后由 **垃圾回收** 回收；块结束、变量被覆盖等会使引用消失，效果上类似「离开作用域不再使用」，但**无**确定性析构顺序。

4. **静态类型**  
   - 若模块中**无**与类名同名的顶层函数，且存在类 **`类名`**，并满足 **其一**：类内有 **`void 类名(…)`**，或类内有 **`New类名`（含由 `New<类名>(…)` 生成）**，则 **`类名(实参…)`** 按所选成员的形参表检查，表达式类型为该**类名**（不透明类型）。

## 指针与引用

> **实现状态：** 已实现取址 `&`、指针类型 `T*`、一元解引用、引用类型 `T&`（别名语义）、以及语句 `*<指针表达式> = <表达式>`。含指针/引用/`&&`/`||` 等路径走 AST 执行而非栈式 ASM。更丰富的指针成员访问（如 `->`）仍以实现为准。

### 指针

#### 取址

对**变量**取址得到指向其所代表对象的指针：

```kval
&<变量名>
```

**限制：** 不能对字面量取址，只能对变量取址。

#### 声明

```kval
<元素类型>* <指针名> = &<变量名>
```

`<元素类型>*` 表示「指向 `<元素类型>` 的指针」类型。

#### 解引用

```kval
*<指针名>
```

在左值语境下可读写被指对象。

#### 多级指针

指针类型可再次作为「被指类型」：

```kval
<元素类型>** <二级指针名> = &<指针名>
```

#### 通过指针访问成员

`->` 与先解引用再 `.` 等价。

读取：

```kval
<指针名>-><成员名>
(*<指针名>).<成员名>
```

写入：

```kval
<指针名>-><成员名> = <值>
(*<指针名>).<成员名> = <值>
```

### 引用

引用是已有对象的**别名**：绑定后，通过引用名的读写等价于对原变量的读写。

#### 声明与初始化

```kval
<元素类型>& <引用名> = <变量名>
```

- `&` 写在**元素类型**之后，表示引用类型。  
- 声明时**必须**初始化；未初始化或非法绑定由编译器报错（以实现为准）。  
- 绑定后引用**不换绑**到其它对象（与指针变量可改指向不同）。

#### 使用引用

无需解引用，语法与同类型对象一致：

```kval
<引用名>                 // 读
<引用名> = <值>          // 写整体
```

#### 成员访问

使用 **`.`**（不是指针的 `->`）：

```kval
<引用名>.<成员名>
<引用名>.<成员名> = <值>
```

#### 与指针用法对照

|  | 指针 `T*` | 引用 `T&` |
| :-- | :-- | :-- |
| 语义 | 存地址，可改指向 | 别名，不换绑 |
| 当作整体用 | `*p` | `r` |
| 成员 | `p->m` 或 `(*p).m` | `r.m` |

#### 限制

- 一般不能将字面量、无名临时量绑到引用（与取址规则一致；若实现有例外以实现为准）。
- 无「空引用」用法；须保证被引用对象在使用期间有效。

## 异常与错误体系

### `throw`

```kval
throw <表达式>;
```

- 抛出目标异常对象；若抛出值不是异常对象，运行时会按实现包装为对应运行时异常。
- 运行器会以 Python 风格的 `Traceback` 版式渲染（`File` / `Lineon` / `In` / 代码上下文 / `ExceptionName: message`），但不直接透出 Python traceback。

### `try-catch-else-finally`

```kval
try {
  <语句...>
}
catch {
  <语句...>
}
catch (ExceptionType) {
  <语句...>
}
catch (ExceptionType e) {
  <语句...>
}
else {
  <语句...>
}
finally {
  <语句...>
}
```

- `catch {}`：捕获所有异常。
- `catch (T) {}`：捕获 `T` 及其子类。
- `catch (T e) {}`：同上，并把异常绑定到 `e`。
- `else` 在 `try` 主体未抛异常时执行；`finally` 总会执行。

### 异常类层次

- 根：`Exceptions`
- 运行时错误：`RunTimeError` 及细分类型（如 `TypeError`、`ValueError`、`NameError`、`IndexError`、`KeyError`、`ZeroDivisionError`、`ImportError` 等）
- 系统错误：`SysError` 及其子类

## 数组声明语法（更新）

数组声明采用以下形态：

```kval
<元素类型> <数组名>[<元素数量>];
<元素类型> <数组名>[<元素数量>] = <初始化表达式>;
```

- `<元素数量>` 需为 `int` 表达式。
- 该语法为数组声明的推荐与规范写法。

## 类继承与 `base`

### 继承语法

```kval
class SubClass : public BaseA, protected BaseB, private BaseC {
  ...
}
```

- 支持多继承。
- 继承列表中的 `public/protected/private` 决定从父类继承成员在子类中的可见性折叠方式。
- 方法查找遵循 MRO。

### `base` 指针

在类方法内可使用 `base` 访问父类成员（按 MRO，从当前类之后开始查找）：

```kval
int f() {
  return base.f();
}
```

- `base` 用于父类成员解析，不等价于 `this`。
- 通过 `base` 对成员写入受限（以运行时访问控制为准）。

## 模块、包与导入导出

### 导入语法

```kval
import ModuleName;
import ModuleName as Alias;
from ModuleName import Member;
from ModuleName import Member as Alias;
```

### 相对导入

```kval
import .ModuleName;
import ..ModuleName;
from .ModuleName import Member;
from ..Package import Member;
```

- `.` 表示当前包层级，`..` 表示上一层，依此类推。

### 导出规则

可通过以下任一方式声明导出：

```kval
const string[2] exports = { "NameA", "NameB" };
```

或在定义时使用 `export` 修饰符：

```kval
export int add(int a, int b) { ... }
```

- 若二者都未提供：默认导出所有不以下划线约定隐藏的公开成员（实现上通常排除 `__` 前缀名）。

### 模块搜索顺序

1. 当前工作目录（`cwd`）
2. `Kval/Lib`
3. `Kval/Lib/site-packages`
4. `Kval/PyModules`

### 包约定

- `PackageName.kval`：包初始化入口。
- `~PackageName.kval`：包析构入口（进程结束时触发）。
- 支持子包与跨包 `from Package import ...`。

## `namespace`

```kval
namespace Name {
  ...
}

Name::member
```

- `namespace` 用于组织符号并提供 `::` 访问。
- 其成员可参与导入与作用域限定解析。

## PythonBridge 强类型约束

Python 模块被 Kval 以桥接模块导入时：

- 导出函数必须为所有参数提供类型注解。
- 导出函数必须声明返回类型注解。
- 未满足时导入阶段直接报类型错误。
